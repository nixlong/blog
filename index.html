​ 可以看到, 在编译阶段:

  • true 直接被翻译为iconst_1 指令, 可当作整型1处理

  • false 直接被翻译为iconst_0 指令, 可当作整型0处理

​ 也就是说, 这3个字面量标识符, 仅用着变量赋值, 在编译阶段结束后, 对应着3条极其简单的处理指令. 因此Java spec 为将其纳入Java语言关键字范畴. 但是这3个词也不能用作变量名出现, 可看作为Java的3个保留字.

Dalvik反编译

[TOC]

1.[注意]不要使用JDK1.8及以上
2.[流程].java -> .class -> .dex –> .smali

##0. Demo

1.javac

编译 .class

2.生成Dex

使用Android dx.jar包将.class转换为.dex, dx.jar一般位于 android-sdk\build-tools* 下面

#Ymodem协议移植

[TOC]

STM32/Dvm项目移植FatFS时,使用W25Qxx Flash做文件系统,不能使用USB或其他外设接口直接进行文件拷贝, 因此移植Ymodem协议用于向Flash下载文件.

##0.Ymodem介绍

Ymodem在嵌入式项目中,常用于文件下载和IAP在线编程.
Ymodem协议是一种发送并等待的协议。即发送方发送一个数据包以后，都要等待接收方的确认，分为Ymodem-1K和Ymodem-G两种子格式.
